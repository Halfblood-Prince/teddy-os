# Phase 9

## Implemented in This Phase

- build-from-scratch guide
- VMware setup and testing guide
- release publishing guide
- updater manifest schema documentation for the deferred updater phase
- guide for integrating future apps/components
- theming/customization guide
- limitations and roadmap document

- `docs/building-from-scratch.md`

## Purpose

The goal of this phase is to make Teddy-OS understandable to a hobby OS
developer working from a fresh checkout. The docs focus on the actual current
state of the repository and clearly separate completed work from deferred work,
especially the skipped updater phase.
